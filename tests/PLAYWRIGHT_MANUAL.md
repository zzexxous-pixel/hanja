## 1. 간략한 핵심 사용법

* **기본 모드 실행:** Playwright는 기본적으로 백그라운드(화면 없음)에서 빠르게 실행됩니다. `npx playwright test`를 입력하면 10개의 시나리오가 순식간에 지나가며 통과 여부를 알려줍니다.
* **디버깅 모드 실행 (Headed):** 브라우저가 진짜 혼자서 버튼을 연타하고 마이크를 켜는지 눈으로 직접 구경하거나 단계별로 디버깅하고 싶다면 명령어 뒤에 `--headed`를 붙여줍니다.
```bash
npx playwright test --headed

```

## 2. 맨 처음 설치부터 설정까지

프로젝트를 처음부터 구축하거나 새로운 컴퓨터에서 환경을 세팅할 때 수행하는 빌드 프로세스입니다.

### Step 1: Playwright 패키지 설치

프로젝트 루트 폴더(HTML 파일이 있는 폴더)에서 터미널을 열고 아래 명령어를 입력하여 무결성 테스트 엔진을 설치합니다.

```bash
npm init playwright@latest

```

* **대화형 옵션 선택 가이드:**
* **TypeScript or JavaScript?** ➡️ `JavaScript` 선택
* **Where to put your end-to-end tests?** ➡️ `tests` (엔터)
* **Add a GitHub Actions workflow?** ➡️ `false` (개인 로컬 검증 중심이므로 N)
* **Install Playwright browsers?** ➡️ `true` (테스트용 전용 브라우저 자동 설치, 엔터)



### Step 2: 환경 설정 파일 수정 (`playwright.config.js`)

설치가 완료되면 루트 폴더에 생성되는 `playwright.config.js` 파일을 열고, 배포 사이트 주소와 크롬 브라우저 환경을 아래와 같이 심플하게 동기화합니다.

```javascript
const { defineConfig, devices } = require('@playwright/test');

module.exports = defineConfig({
  testDir: './tests',
  fullyParallel: true,
  reporter: 'html',
  use: {
    // 💡 테스트가 바라볼 깃헙 페이지 대상 주소 고정
    baseURL: 'https://zzexxous-pixel.github.io/hanja-test/', 
    trace: 'on-first-retry',
  },
  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },
  ],
});

```

## 5. 초보자를 위한 상황별 트러블슈팅 (테스트 실패 대처법)

제미나이와 코드를 고치다 보면 반드시 테스트가 실패(빨간불)하는 순간이 옵니다. 이때 당황하지 말고 아래 3가지 상황에 맞게 대처하세요.

* **상황 1: 기능은 잘 되는데 디자인 스냅샷(시나리오 1, 4)만 빨갛게 실패할 때**
* **원인:** 한자 카드의 여백을 바꾸거나 폰트를 수정하는 등 **의도적으로 디자인을 변경한 경우**입니다. Playwright는 기존 기준 이미지와 픽셀 하나만 달라져도 에러로 인지합니다.
* **해결법:** 내가 의도해서 바꾼 디자인이 맞다면, 터미널에 아래 명령어를 쳐서 현재 바뀐 화면을 새로운 '정상 기준'으로 업데이트해 주면 됩니다.
```bash
npx playwright test --update-snapshots

```




* **상황 2: 깃허브 레포지토리 이름이 바뀌거나 배포 주소(URL)가 달라졌을 때**
* **원인:** 테스트 시작 시 접속할 기본 주소가 일치하지 않아 페이지를 로드하지 못하는 경우입니다 (`Cannot navigate to invalid URL`).
* **해결법:** `playwright.config.js` 파일을 열고 `use: { baseURL: '새로운 깃헙 주소' }` 부분만 수정해 주면 바로 정상 작동합니다.


* **상황 3: 10개 중 특정 시나리오가 갑자기 실패할 때**
* **원인:** 제미나이가 코드를 리팩토링하면서 HTML의 클래스명(`theme-emerald` 등)을 지웠거나, 클릭 이벤트 위임 함수(`data-action`)를 실수로 누락시킨 경우입니다.
* **해결법:** 먼저 `npx playwright show-report`를 입력해 리포트를 띄우고, 어떤 엘리먼트를 찾지 못했는지 확인한 뒤 "제미나이야, 코드를 고치다가 스크립트에서 이러이러한 데이터 속성이 누락됐어. 테스트 통과하게 다시 복원해줘"라고 수정 요청을 보내면 됩니다.
