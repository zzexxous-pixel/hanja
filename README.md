# 4급 한자 마스터 플랫폼 세부 기능 명세서 (Functional Specification)

## 1. 시스템 개요
본 명세서는 순수 Vanilla JS 및 GPU 가속 기반의 저사양 타깃 배정한자 학습기의 전산 인터랙션 시스템 무결성을 검증하고, 향후 기능 확장 및 QA(품질 보증)의 기준점으로 삼기 위한 표준 개발 가이드라인이다.

## 2. 화면 레이아웃 및 UI 컴포넌트 규격
### 2.1 헤더 컨트롤 타워 (Sticky Header)
- **로고 및 타이틀 영역**: '4급 한자 마스터' 텍스트 표기. 마우스 오버 시 일반 화살표 커서(\`cursor-default\`)를 유지하여 오인 조작을 방지한다.
- **폰트 스케일러**: [A-] 및 [A+] 버튼 배치. 터치 딜레이 0ms 유틸리티(\`fastTouch\`)와 연동되어 작동하며, 한자 크기(16px~64px) 및 훈음 크기(9px~28px)를 \`localStorage\`와 실시간 동기화한다.
- **페이지 스위처 (Pager)**: 일반 모드(1~6 페이지) 구동 시 삼각형 페이저 및 현재 페이지 인디케이터(\`1 / 6\`)가 활성화된다. 즐겨찾기 진입 시 오동작 방지를 위해 반투명(\`opacity-50\`) 및 이벤트 잠금 처리(\`pointer-events-none\`)를 수행한다.
- **자가 테스트 (훈음 가리기) 토글**: 가로 너비 \`w-28\`(112px) 및 중앙 정렬(\`justify-center\`)이 엄격히 고정된 둥근 사각형 버튼. 활성화 시 초록 테마(\`bg-emerald-600\`), 비활성화 시 노란 테마(\`bg-yellow-500\`)로 스왑되며, 상태 전환 시 버튼 너비가 1px도 변하지 않는 레이아웃 고정 구조를 유지한다.

## 3. 제스처 및 핵심 인터랙션 생명주기
### 3.1 일반 모드 (isQuizMode === false)
- 한자 카드 영역 탭(클릭/터치) 시 즉각 상세 보기 모달 팝업(\`openModal\`)을 호출한다.
- 우측 상단 별표(★) 클릭 시 즉각 \`localStorage\` 배열에 인덱스를 가감하고, 가상 메모리 정적 탭 캐시(\`tabCache\`) 내부 노드까지 실시간 양방향 UI 바인딩을 완수한다.

### 3.2 훈음 가리기 모드 (isQuizMode === true) - 대책A 및 대책1 융합 시스템
- **터치 시작 (\`touchstart\` / \`mousedown\`)**: 
  - 기본 브라우저 스크롤을 방해하지 않기 위해 \`e.preventDefault()\`를 호출하지 않으며, 네이티브 스크롤 스레드를 보존한다.
  - 최초 터치 시점의 유클리드 좌표(\`clientX\`, \`clientY\`)를 메모리에 캡처하고 300ms 정밀 타이머를 가동한다.
  - 보안 제약 우회를 위해 동기 컨텍스트 내부에서 Web Speech API(\`recognition.start()\`)를 즉시 시동한다.
- **터치 이동 (\`touchmove\` / \`mousemove\`)**:
  - 첫 터치 좌표 대비 손가락 변위차가 **10px를 초과**하면 화면 스스크롤 상태로 확정한다.
  - 스스크롤 확정 즉시 대기 타이머를 파괴하고 마이크 오디오 수집을 강제 취소(\`recognition.abort()\`)하여 자바스크립트 스레드를 네이티브 스스크롤 가속 모드로 완전 이관한다.
- **터치 종료 (\`touchend\` / \`mouseup\`)**:
  - **누른 시간이 300ms 미만이고 이동 변위가 10px 이내인 경우 (단순 탭)**: 음성 수집을 취소(\`abort\`)하고 상세 보기 모달 팝업(\`openModal\`)을 띄운다.
  - **누른 시간이 300ms 이상 유지된 경우 (정상 홀드 학습)**: 시각적 카드 펄스 효과를 즉시 종료시키고, 짧은 단어(\`뿔 각\` 등)의 받침이 잘리는 현상을 방지하기 위해 **400ms의 잔음 꼬리 버퍼(Tail Time Buffer)**를 확보한 뒤 \`recognition.stop()\`을 호출하여 최종 오디오 분석 스레드로 진입시킨다.

## 4. 음성 인식 및 발음 유사도 판정 알고리즘
- **자모 분해 엔진**: 입력된 음성 데이터와 배정한자 정답 텍스트를 유니코드 범위(\`44032 ~ 55203\`) 내에서 초성, 중성, 종성 매핑 테이블을 거쳐 물리적인 순수 자모 문자열 문자열로 완전히 해체한다. 모든 전치/후치 공백 및 기호는 정규식(\`/[\\.\\?\\!\\,\\s]+/g\`)으로 정제한다.
- **레벤슈타인 거리 (Levenshtein Distance)**: 해체된 두 자모 문자열 간의 삽입, 삭제, 대체 비용을 연산하여 최적의 편집 거리를 도출한다.
- **합격 임계값**: 자모 일치 점수가 **60%(\`0.6\`) 이상**일 경우 최종 정답으로 판정한다.

## 5. 저사양 디바이스 최적화 및 디버그 시스템
- **하드웨어 가속 그래픽**: 저사양 칩셋의 GPU 부하를 원천 차단하기 위해 무거운 \`box-shadow\`(그림자 번짐) 그래픽 연산을 전면 배제하고, 하드웨어 가속이 용이한 \`border-color\` 및 \`background-color\` 기반의 펄스 키프레임 애니메이션(\`mic-border-pulse\`)으로만 구동한다.
- **비밀 디버그 콘솔 이스터에그**: 헤더 제목 영역을 **2.5초 이내에 5회 연타**하면 최하단에 고정 폭 터미널(\`z-50\`)이 슬라이드 업된다. 메모리 한계 조율을 위해 순환 큐 메커니즘을 적용하여 최대 50개의 로그만 스택에 유지하고 올드 로그는 자동 소멸시킨다.
